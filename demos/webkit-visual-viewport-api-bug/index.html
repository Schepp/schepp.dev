WebKit Visual Viewport API Bug

WebKit has a bug where, when pinch-zooming, the results of visualViewport.width × visualViewport.scale and visualViewport.height × visualViewport.scale do not correspond to the unzoomed visual viewport's* width and height.

QR-Code of this page for easier testing on mobile:

When is this important?

When you have a fixed element that you place within the boundaries of the unzoomed visual viewport, so that it is never overlapped by the soft keyboard.

Pinch-zooming in WebKit results in jumping and wiggling of that element during the zoom process, as Visual Viewport API data used for calculations is inaccurate.
